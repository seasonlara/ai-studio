# 扣子编程直接开发指令

请开发一个“台湾虾皮电商 AI 生图工具”的最小可用版本。  
核心目标：用户上传 1 张产品白底图，系统自动生成一套台湾虾皮风格的商品主图 9 张，以及详情页 12 张。

不要先开发登录、支付、积分、会员、后台管理、长期图片存储。第一版只做核心生图流程。

## 一、页面功能

页面只需要一个主工作区，包含以下内容：

1. 产品标题，必填。
2. 上传产品图，必填，至少 1 张，最多 8 张。
3. 生成内容切换：
   - 主图生成
   - 详情页
4. 模型选择：
   - Banana Pro
   - Seedream 5.0
   - GPT Image 2
5. 模式选择：
   - 标准极速：一次生成整套图。
   - PRO 增强：用户可选择张数、比例、产品功能、画面风格。
6. 输出语言：
   - 默认繁体中文（台湾）。
7. 详情页批次：
   - 仅在用户选择“详情页”时显示。
   - 选项为“前6屏”和“后6屏”。
8. 生成按钮。
9. 生成结果区：
   - 默认隐藏。
   - 用户点击生成后才显示。
   - 图片生成中不要用用户上传图占位，要显示独立的生成中状态。
   - 图片完成后显示图片卡片、图片名称、下载、重新生成按钮。
   - 点击图片可放大查看。
   - 重新生成只重生成当前这一张，不要重生成全部。
   - 支持一键打包下载当前结果列表。

刷新或关闭页面时，如果页面已有生成结果，需要弹窗提醒用户确认，避免结果丢失。

## 二、用户输入

必填：

```json
{
  "product_title": "产品标题",
  "product_images": ["用户上传的产品图"]
}
```

可选：

```json
{
  "product_function": "用户补充的产品功能卖点",
  "material": "材质",
  "size": "尺寸",
  "target_audience": "适用人群",
  "visual_style": "用户偏好的画面风格",
  "extra_info": "其他补充信息"
}
```

如果用户没有填写可选项，AI 自动分析图片并推断，但不能编造硬参数。

硬参数包括：年龄、体重、承重、尺寸、材质、认证、检测标准、防水等级、适配车型、保固年限、配件清单。

无法确认时不要写具体数字，改用：

- `以賣場資訊為準`
- `內容物依實際出貨為準`
- 或直接省略。

## 三、整体生图流程

实现流程必须分 5 步：

1. 读取用户上传的产品图。
2. 调用视觉分析模型，识别产品品类、颜色、材质、结构、卖点、使用场景、目标人群。
3. 输出结构化商品分析 JSON。
4. 根据商品分析自动匹配视觉风格，并生成每张图的独立提示词。
5. 逐张调用生图模型生成图片。

不要把 9 张主图或 12 张详情图塞进一个提示词里一次生成。必须逐张生成，这样才能控制主题、重试失败图片、保持命名一致。

## 四、商品分析 JSON

生图前先让 AI 输出如下结构：

```json
{
  "product_category": "产品品类",
  "core_product_name": "核心产品名",
  "color_style": "产品颜色和视觉气质",
  "visible_materials": ["图片中可见材质"],
  "visible_parts": ["图片中可见结构"],
  "visible_features": ["图片中可确认功能"],
  "likely_scenarios": ["合理使用场景"],
  "target_audience": ["目标人群"],
  "risk_unknowns": ["不能确认的信息"]
}
```

要求：

- 只能基于图片和用户输入分析。
- 必须保留产品真实颜色、形状、比例、结构。
- 必须区分“可见卖点”和“不能确认的信息”。
- 不能新增不存在的配件、品牌、结构、认证、参数。

## 五、自动匹配视觉风格

前端不要让普通用户选择复杂平台风格。默认显示：

```text
Shopee AI 自动匹配
```

系统根据产品自动选择风格。

### 1. 黑色科技车用品

适用：

- 车载手机支架
- 磁吸支架
- 真空吸附支架
- 黑色金属数码配件
- 车载收纳件

视觉：

- 黑色、枪灰、金属质感
- 冷蓝、冰白、深色科技背景
- 车内中控、仪表台、挡风玻璃
- HUD 光效、发光吸附圈、工程线条

文案：

- 真空吸附
- 稳固防抖
- 360度旋转
- 折叠收纳
- 车内适用
- 精密质感

禁止编造：

- N52
- 120N
- 5kg
- 航空级
- 专利
- 任何未提供硬参数

### 2. 可爱居家小物

适用：

- 迷你封口机
- 塑封机
- 零食保鲜器
- 小型厨房工具
- 可爱造型小家电

视觉：

- 奶油色、粉色、浅黄
- 木纹桌面
- 下午窗光
- 软萌圆角标签
- 可爱插画元素

文案：

- 一压即封
- 零食保鲜
- 轻巧便携
- 磁吸收纳
- USB 充电
- 防潮防霉

### 3. 亲子安全用品

适用：

- 儿童座椅
- 儿童安全带辅助垫
- 婴幼儿出行用品
- 亲子车用品

视觉：

- 明亮车内或家庭场景
- 蓝白、暖橙、柔和背景
- 信任感图标
- 结构标注清晰

文案：

- 稳固支撑
- 透气舒适
- 多点固定
- 亲肤面料
- 便携安装
- 可拆清洗

禁止编造：

- 认证
- 适用年龄
- 适用体重
- 碰撞保护
- 承重

### 4. 美妆保养健康感

适用：

- 精华
- 牙膏
- 口腔护理
- 清洁护理
- 香膏除臭

视觉：

- 白色、金色、浅蓝
- 干净高级
- 玻璃反光
- 成分感、清洁感

禁止：

- 医疗疗效
- 治疗承诺
- 保证有效

### 5. 通用橙色爆款

当系统无法判断更适合的风格时，使用通用 Shopee 爆款风格：

- 橙色、奶油色、浅暖背景
- 粗黑繁体中文大字
- 白色描边
- 卖点标签
- 圆角卡片
- 箭头、放大镜、对比图

## 六、9 张主图结构

主图生成必须按以下 9 张规划：

1. `主图 01｜全方位主視覺`  
   场景 + 核心卖点。第一张最重要，要美观、突出、商业感强。必须展示产品主体、使用场景、3-5 个核心卖点。

2. `主图 02｜核心功能賣點圖`  
   聚焦最强卖点，例如真空吸附、一压即封、稳固支撑、透气舒适。

3. `主图 03｜折疊多角度圖`  
   展示正面、侧面、折叠、展开、多角度结构。

4. `主图 04｜升級比較圖`  
   本商品 vs 一般商品。只能比较可确认或合理的结构差异，不得编造认证和参数。

5. `主图 05｜商品規格表`  
   商品名称、颜色、材质、尺寸、重量、适用范围。未知项写 `以賣場資訊為準`。

6. `主图 06｜細節特寫圖`  
   3-4 个细节特写，必须和原图结构一致。不要换成其他产品的扣具、接口或材质。

7. `主图 07｜包裝內容圖`  
   展示产品主体和配件。配件未知时写 `內容物依實際出貨為準`，不要编造说明书、线材、安装包。

8. `主图 08｜使用情境圖`  
   展示真实使用场景，例如车内、厨房、桌面、浴室、户外、亲子出行。

9. `主图 09｜信任促銷圖`  
   展示颜色、版本、质感、适用场景或促销信任感。不能编造保固、认证。

## 七、12 张详情页结构

详情页分两批生成。

### 前6屏

1. `詳情 01｜功能總覽`  
   产品主视觉 + 4 个功能图标。

2. `詳情 02｜痛點共鳴`  
   展示用户痛点，例如传统产品笨重、不稳、占空间、麻烦。

3. `詳情 03｜解決方案`  
   展示本产品如何解决痛点。

4. `詳情 04｜功能矩陣`  
   展示多个功能点，类似功能矩阵。

5. `詳情 05｜核心結構`  
   标注产品结构、受力点、关键部件。

6. `詳情 06｜規格概覽`  
   展示规格表，未知参数必须保守处理。

### 后6屏

7. `詳情 07｜便攜賣點`  
   展示轻巧、便携、随取随用。

8. `詳情 08｜形態展示`  
   展示折叠、展开、收纳或多角度。

9. `詳情 09｜舒適體驗`  
   展示使用体验、亲肤、透气、轻松。

10. `詳情 10｜傳統痛點`  
   展示一般商品的不足。

11. `詳情 11｜穩固受力`  
   展示固定、受力、稳定、防滑。

12. `詳情 12｜規格補充`  
   展示最终规格、注意事项、内容物补充。

## 八、提示词通用模板

每张图生成前都要用这个模板组装提示词：

```text
请基于上传的产品参考图，生成一张台湾 Shopee 电商商品图。

产品标题：{product_title}
核心产品名：{core_product_name}
产品品类：{product_category}
当前图片主题：{image_title}
当前图片目标：{image_goal}
用户补充功能：{product_function}
用户偏好画面风格：{visual_style}

画面要求：
- 生成 1 张完整电商图，不要拼成多张独立图片。
- 保持产品外观、颜色、比例、结构与上传图一致。
- 使用繁体中文，不要使用简体中文。
- 文案要适合台湾消费者和 Shopee 上架。
- 字体要大，手机端可读。
- 同一组图片保持统一色调、字体、描边、按钮、标签和背景风格。
- 不要出现水印。
- 不要出现乱码、错字、英文占位符。
- 不要编造认证、重量、尺寸、承重、保固、适配车型。

视觉风格：
{matched_style_description}

本张图片必须包含：
{required_elements}

本张图片禁止：
{forbidden_elements}
```

## 九、一组图的风格一致性

生成一组图之前，先生成一个 `style_lock`，然后每张图都带上它：

```json
{
  "palette": "主色、辅助色、背景色",
  "typography": "字体粗细、描边、阴影",
  "layout": "标题位置、标签样式、卡片圆角",
  "background": "场景和光线",
  "accent": "按钮、箭头、图标、光效"
}
```

要求：

- 同一批 9 张主图必须统一风格。
- 同一批 6 张详情图必须统一风格。
- 不能出现其中一张突然变成完全不同色调。
- 如果用户输入画面风格，以用户输入为优先。
- 如果用户没有输入，AI 根据产品自动匹配。

## 十、模型调用建议

每张图单独调用生图模型。

推荐处理方式：

- Banana Pro：适合商业主图，建议并发 2-3 张。
- Seedream 5.0：适合产品保真，建议并发 1 张，并关闭水印。
- GPT Image 2 Edit：适合基于上传图做强参考编辑。
- GPT Image 2：适合文生图或轻参考场景。

不要在前端暴露真实 API Key。API Key 必须放在服务端环境变量里。

## 十一、输出命名

如果用户输入产品标题，例如 `磁吸手机支架`，图片名称应为：

```text
磁吸手机支架-主图 01｜全方位主視覺
磁吸手机支架-主图 02｜核心功能賣點圖
磁吸手机支架-詳情 01｜功能總覽
```

下载文件名必须和页面展示名称一致。

一键下载压缩包命名：

```text
产品标题 + 主图/详情图 + 图片数量 + 下载日期
```

示例：

```text
磁吸手机支架-主图-9张-2026-06-30.zip
```

## 十二、质量检查

每张图生成后检查：

1. 是否为繁体中文。
2. 是否有乱码或错字。
3. 产品是否和原图一致。
4. 是否编造了硬参数。
5. 图片主题是否符合当前编号。
6. 同一组图片风格是否统一。
7. 是否出现水印。
8. 是否适合台湾 Shopee 手机端浏览。

如果某张失败，只允许重试该张，不要重跑整组。

## 十三、第一版验收标准

第一版完成后必须满足：

1. 用户上传 1 张产品图并填写标题后，可以生成 9 张主图。
2. 用户切换到详情页后，可以选择前 6 屏或后 6 屏生成。
3. 主图和详情页结果互不清空。
4. 单张图片可放大、下载、重新生成。
5. 支持一键下载当前结果。
6. 刷新或关闭页面前，如有生成结果，弹窗提醒。
7. 无真实 API Key 出现在前端代码、日志、文档或截图里。
